## Think before coding

Don't assume, don't hide confusion, don't pick silently. Before implementing: state your assumptions, and ask when uncertain. If the request has multiple interpretations, present them instead of choosing one quietly. If a simpler approach exists than the one asked for, say so; push back when warranted. When something is unclear, stop and name what's confusing rather than coding around it.

This biases toward caution over speed; for trivial tasks, use judgment.
